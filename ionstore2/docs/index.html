D) WooCommerce - top

WooCommerce is a free eCommerce plugin that allows you to sell anything, beautifully. Built to integrate seamlessly with WordPress, WooCommerce is the world’s favorite eCommerce solution that gives both store owners and developers complete control.

With endless flexibility and access to hundreds of free and premium WordPress extensions, WooCommerce now powers 30% of all online stores -- more than any other platform.

D.1) Installation - top

If you already have Wordpress and WooCommerce installed, then just skip this step.

If you have web hosting already and would like to install WooCommerce using the Wordpress Admin, follow this step:

  1. Login to your Wordpress Admin panel
  2. Go to Plugins -> Add New
  3. Type "WooCommerce" in the search field and hit enter. Wait until it's found
  4. Click Install Now
  5. Wait until WordPress completes the installation
  6. After the installation finish, configure the WooCommerce

Once WooCommerce is activated, the WooCommerce Onboarding Wizard can help you set up your shop. Select Let’s Go! to get started. Or select Not Right Now if you prefer to manually set up your WooCommerce shop.

Reference: Installing and Uninstalling WooCommerce

D.2) Import Dummy Data - top

WooCommerce provide dummy data products and categories for your eCommerce store. You can use it to see how the app app works with WooCommerce.

Dummy data for WooCommerce is located in a file called dummy-data.xml, which is located in the WooCommerce plugin directory. Once WooCommerce is installed, you can find it three ways:

  • Use your cPanel or FTP, go to 'wp-content/plugins/woocommerce/dummy-data' and locate the dummy-data.xml file
  • Unpack or unzip the WooCommerce .zip file
  • Download from WooCommerce Github

Importing the WooCommerce Dummy Data

  1. Login to your Wordpress Admin panel
  2. Go to Tools -> Import. Select WordPress.
  3. If you do not have the WordPress importer installed, you will be prompted to install the importer and then proceed. You have the choice of installing manually or clicking the Install Now button. Select the Choose a file from your computer button and then the dummy-data.xml file you located above.
  4. You will be presented with a screen to import or select an author and various other options related to download and import file attachments
  5. Select who you want to make the author of the new products from the dropdown, such as your admin user, and Select the download and import file attachments checkbox. The latter will download all the images to your local server for use on the products
  6. Click Submit and your dummy data will be installed. You now have a working copy of WooCommerce with data.

Reload and re-run your app to see the dummy data.

Reference: Importing WooCommerce Dummy Data

D.3) Shipping - top

You can add multiple shipping method from WooCommerce and your apps. To add shipping, please follow these step:

  1. Login to your Wordpress Admin panel
  2. Go to WooCommerce -> Settings -> Shipping.
  3. Select Flat Rate shipping method and click Add shipping method
  4. Input your cost. Copy and save the instance_id variable from the URL
  5. Repeat the process above to create the Worldwide Flat Rate
  6. Input your cost. Copy and save the instance_id variable from the URL
  7. You can also add other shipping method (Free Shipping or Local Pickup)
  8. Open your js/app.js file to configure the shipping method and id

Save and run your app on browser to see how the shipping method works.

D.4) Payment - top

Now, you have to active the payment method for your WooCommerce. To active the payment, please follow this step:

  1. Login to your Wordpress Admin panel
  2. Go to WooCommerce -> Settings -> Checkout
  3. Enable the BACS settings. And input your bank account. Your bank account will be shown in the buyer email invoice
  4. Enable the PayPal settings and input your PayPal receiver email. You can thick the Sandbox mode when in development and un-thick the Sandbox when your WooCommerce is ready to publish
  5. Go to WooCommerce -> Settings -> Checkout

Your PayPal is ready now in WooCommerce. Now, lets setting up PayPal for your mobile apps.

D.5) Rest API - top

In order to get app works with your WooCommerce website, you have to activate the Rest API of WooCommerce. Please follow this step to do it:

  1. Login to your Wordpress Admin panel
  2. Go to WooCommerce -> Settings -> API. Enable the REST API.
  3. Click Add Key button
  4. Input the Description, User and set the Permission to Read/Write. Then click Generate API Key
  5. Copy the Consumer Key and Consumer Secret into the js/ap.js at constant variable (variable ConsumerKey & ConsumerSecret)

Now, go to next section.

E) JSON API & JSON API User Plugins - top

JSON API & JSON API User are free wordpress plugin. We need these two plugins in order to allow user signup and signin in our app. Every user that has woocommerce account in you website, they can sigin also in your app. New customer can also signup from app.

To install these plugins, please follow these step:

  1. login into your WordPress Admin. Go to Plugins menu and Add New
  2. Type "JSON API" in the search field and hit enter. Wait until it's found
  3. Click Install Now and wait until WordPress completes the installation
  4. Active the JSON API plugin
  5. Go to the Plugin menu again and repeat the process for "JSON API user" plugin
  6. Now, we need to active the API so it can works with our app
  7. Go to Settings and click JSON API
  8. Make sure that the API User status is in Active position.

Now, your plugins are ready to work.

G) OneSignal Push Notification - top

OneSignal is a high volume and reliable push notification system for mobile and web applications. We support all major native and mobile platforms by providing dedicated SDKs for each platform, a RESTful server API, and an online dashboard to send push notifications.

To use OneSignal with your app, firstly create Google API and then create account in OneSignal website.

G.1) Google API Registration - top

We have to create an app in Google Developer to get the client ID that we will use in sending the push notification. Follow these step:

Let's create our app on Google Developer and setup the GCM (Google Cloud Messaging) to send the notifications.

  1. Visit the Google Service Wizard
  2. Type any name to create a new app, or select an existing Google app from the dropdown. In this example, we create an app named “Ionic Premium”
  3. Click Choose and configure services to continue
  4. Select Cloud Messaging and then Click enable Cloud Messaging
  5. It will show the server API key and GCM Sender ID
  6. Add this GCM Server API key into OneSignal and GCM Server ID key into your js/app.js in the constant variable (variable GCM_SenderID)

G.2) OneSignal Android Configuration - top

In order to get OneSignal works on your apps, please create your apps on OneSignal by following these steps:

  1. Login to your OneSignal account
  2. Create new app in OneSignal, type your app name and click Create
  3. Select Google Android (GCM) to configure and click Next
  4. Type your GCM Server API key (from G.1. Google API Registration) and click Save
  5. Select Phonegap, Cordova, Ionic, Intel XDK and click Next
  6. It will show your App ID. Copy this OneSignal App ID into your js/app.js in the constant variable (variable OneSignalAppID). Then run your app on real android device. When your run app successfully on device, click Check Subsribed Users to make sure that your OneSignal works with that device. Then click Done.
  7. Congratulations! Your push notification is ready to use.

Now, every Android device that install your app will appear in yoru OneSignal Dashboard. You can easily send them push notification from this dashboard directly or by schedule.

Note: Make sure that your device is connected with Internet connection when configuring OneSignal.

G.3) Send Push Notification - top

It is very easy to send the notification to your customer. The notification can be promo code, new product release, useful info from your business or some useful info to your valuable customer. Please follow this step:

  1. Login to your OneSignal account
  2. Click New Message from left menu. Then type your push Title and Content.
  3. If you want to use big picture as your push content, you can scroll below and input the image url Big Picture field.
  4. Scroll to bottom and click the Preview button.
  5. You will see the summary of push content. Click Send button to send the notification.

Then you will see the notification from each of device that install you app.

H) PayPal - top

PayPal is one of the most popular payment gateway in eCommerce industry nowadays. We concern about this and bringing this method into your eCommerce app. You need to have a PayPal account to use this feature. Create a new one, if you don't have yet. To create PayPal account, go to www.paypal.com.

H.1) Sandbox Account - top

Sandbox Account are your default BUSINESS and PERSONAL account to make a demo transaction. Business indicate the merchant account and personal indicate the buyer account.

To change the password, profile, balance of Sandbox account, you can click the record.

H.2) Create App & Sandbox Mode - top

To get our apps work with PayPal. We need to create app (REST API) from PayPal. This Apps will generate Client ID that we used in our mobile app. We will used Sandbox mode to test the PayPal with our app. While using Sandbox, PayPal will not charge your balance. To create app in PayPal and active Sandbox, follow this step:

  1. Login to your PayPal Developer account
  2. Go to My Apps & Credentials and click Create App button
  3. Type your App Name. Then the Sandbox developer account is your default PayPal business account for Sandbox. Now click Create App. button
  4. The Client ID will be shown on your screen. Make sure the Sandbox button at the top right is in active position
  5. Copy your Client ID into src/app.global.ts file (variable paypalSandboxClientID)

Now you can use PayPal Sandbox in your app. Re-run your app in Android. When you make an order, choose PayPal as your payment method. Then, on the login screen. Use your default Sandbox personal account (for me, it is mr.ilhami-buyer@gmail.com). So, this Sandbox mode will demonstrate how the PayPal works when a buyer pay for their order for the merchant.

To view the transaction history, you can go to Sandbox Transactions menu in the PayPal Developer Dashboard.

H.3) Live/Production Mode - top

To use Live Mode, you can repeat the above process by selecting/activating Live button first. Then copy the Client ID the constant variable in your src/app.global.ts file (variable paypalLiveClientID).

To view the transaction history, you can go to Live Transactions menu in the PayPal Developer Dashboard.

I) Getting Started - top

Please follow these step to get the app work. You can also watch our video tutorial below to get started.

I.1) Installing Ionic, Cordova, NPM and Node Modules - top

To run this Ionic, you have to install Ionic Framework by following this step. Download and install devendable version of node.js. Then open your cmd.exe for windows or Terminal for mac and run:

			npm install -g cordova@latest ionic@latest
			npm install -g npm@latest
			npm update

Or if you use mac, don’t forget to use "sudo":

			sudo npm install -g cordova@latest ionic@latest
			sudo npm install -g npm@latest
			npm update

Ionic and cordova are now ready on your computer. No we need to re-install node_modules into the app. Go to iontore2/app

			npm i

Note: Make sure your internet connection is stable and running well. It may take several minutes to install all modules, wait till all done. Now you need to setup your app with your WooCommerce API key.

I.2) Ionstore Configuration - top

Ionstore 2 is very easy to setup, you only need to edit the src/app/app.global.ts and/or config.xml (please edit this with yours) our app will do the rest. Follow this video to setup your Ionstore 2.

Now your app ready to works in browser, by using:

			ionic serve --lab

Now, you app is on chrome browser and running well.
Note: All of hardware plugin will not work on browser.

I.5) Re-Install Plugins - top

After making sure your app running well on browser and android SDK has been installed, now we need to run it on Android and iOS (need mac).

  1. To prepare your app for Android, run this: 
    					sudo cordova prepare android
  2. If you use mac, then you can prepare both android and ios at the same time, run this: 
    					sudo cordova prepare
  3. Note: Make sure your internet connection is stable and running well. Wait till all process done and make sure no error occurs
  4. If you are facing any errors, run npm cache clear and then re-run the instructions above

Those instructions are used to re-install all plugins that will used in the app. We have already declared it in package.json file, so you only need to re-install them using "cordova prepare".

I.6) Build and Run on Android - top

To build your project to Android platform, please make sure that you have completed the instructions above. After the installation complete, you can build the project using:

			ionic cordova build android

This process will produce apk file on your platforms/android.

Now, let’s start running & testing on real device. Follow this step:

  1. Connect USB cable on your Android device to computer (Windows only)
  2. Run the app on real device using: 
    					ionic cordova run android
  3. If you want to run Android on Mac, you can use android emulator or Genymotion
  4. Emulate the app using: 
    					ionic cordova  emulate android

You will see the app running on your real device or android emulator.

I.7) Build and Run on iOS - top

To build your project to iOS platform, open your Terminal and type:

			ionic cordova build ios

The build process will produce ios folder in your platforms folder. Then, to run your app on emulator, please follow this step:

  1. Open Ionstore2/platforms/ios/ionstore2.xcodeproj
  2. Double click ionstore2.xcodeproj file, it will open Xcode
  3. Your can run your app by selecting iOS device or device emulator, then click play button
  4. You can also run the app using command line from Terminal, using 
    					ionic cordova emulate ios
  5. To emulate on specific iOS device (eq. iPhone 5s) 
    					ionic cordova  emulate ios --target="iPhone-7s"

You will see the app running on iOS simulator.

2. How to change the icon and splash screen?

To change Icon and Splash Screen, you can do it by following this step:
  1. Create your own icon.png and splash.png, save it in resources directory. Change all of default icon file with your own icons file. The icon images dimension should follow the Ionic Framework recommended icon file dimension. This provide the recommended size icon files.
  2. Or use these icon and splash screen template.
  3. Open your cmd or Terminal, then: 
    						ionic cordova resources

  4. Every time you change the icon or splash screen, re-run ionic cordova resources

5. Why app can not create order? Error 401?

To get it works, add "content-type" in Access-Control-Allow-Headers (in .htaccess file). So it will be: 
			header set Access-Control-Allow-Headers "origin,content-type,authorization,user-agent,accept,x-r$"

7. Error: linker command failed with exit code 1 (use -v to see invocation) in XCODE

8. Error: There are multiple entries in the deeplink
